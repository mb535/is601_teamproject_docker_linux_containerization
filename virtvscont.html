Virtualization vs Containers!


Virtualization and containers have brought along a revolution. Virtualization has been in existence for years and containers are a relatively newer phenomenon, however, they've changed how severs and application are implemented. Together, they ensure newer advances in customer and user experiences and make the job easier for all involved, from developers, to testers, to analysts, to end users and everyone in between.


How do the two compare, though? For a refresher, virtualization is a virtual machine containing everything from an operating system to all applications that is run on top of and managed by a hypervisor. Containerization is a "container" that contains one (or more) application and all it's dependencies that are required to successfully run and operate. A container is installed and managed by a container engine which provide the interface to the host OS and provide the application in the container all the services it needs.



From the refresher, it is evident that containers are lightweight than virtual machines, owing to the fact that they do not contain the host OS in the image and instead rely on the container engine to interface with the host OS. Since the container engine is available on multiple OSs, the container can thus be run regardless of the host OS.
